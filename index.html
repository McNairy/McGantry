Features

Everything a platform team needs

Built for engineering orgs that want great developer experience without the operational complexity.

Service Catalog

A single source of truth for every service, API, team, and infrastructure component. Relationships, ownership, and dependencies — all in one place.

Single Binary

No Kubernetes, no Node.js, no microservices. One Go binary with embedded SQLite. Deploy it on any server in minutes — or inside Docker.

Plugin Ecosystem

Kubernetes, GitHub, and ArgoCD plugins ship out of the box. Sync entities, view live deployments, stream logs, and trigger syncs — all inside the catalog.

Self-Service Actions

Define runbooks as executable actions with JSON Schema forms. Let developers provision environments and trigger workflows — without opening a ticket.

Full-Text Search

Find any service, team, API, or doc in milliseconds. Powered by SQLite FTS5 with smart ranking — no Elasticsearch cluster required.

GitOps Native

Define your catalog in YAML and apply it with gantry apply. Every entity is version-controlled in Git.

How it works

Up and running in minutes

No infrastructure expertise required. If you can run a binary, you can run Gantry.

01

Install the binary

Download a single pre-built binary for Linux, macOS, or Windows. Or pull the Docker image — your choice.

# macOS / Linux curl -sSL https://github.com/go2engle/gantry/releases/latest/download/install.sh | sh # Docker docker run -p 8080:8080 ghcr.io/go2engle/gantry:latest
02

Define your catalog

Write entity YAML files for your services, APIs, teams, and infrastructure. Schema validation and auto-complete work out of the box.

# catalog.yaml apiVersion: gantry.io/v1 kind: Service metadata: name: payment-api labels: team: payments spec: description: "Handles all payment processing" lifecycle: production repoUrl: https://github.com/acme/payment-api
03

Apply and explore

Push your catalog to Gantry with one command. Your team can immediately discover, navigate, and act on all your infrastructure.

gantry apply -f catalog.yaml ✓ Created Service/payment-api ✓ Created Team/payments ✓ Created API/payment-api-v2 open http://localhost:8080
